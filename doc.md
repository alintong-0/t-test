作为资深游戏系统策划，我已将《消除大龙》核心系统的全套策划方案整合重构。本方案聚焦于**“有限缓存驱动的异步队列消除”**机制，旨在提供一份逻辑严密、表现清晰、可直接对接开发的工程化文档。

# 《消除大龙》核心消除系统策划方案

## 第一阶段：系统拆解与底层抽象

### 1.1 核心循环 (Core Loop)

* **输入端：** 玩家从棋盘点击带有数字（耐久度）的毛线球。
* **中转端：** 毛线球进入有限的缓存槽（Slot）。
* **消除端：** 槽内毛线球实时匹配“大龙”队列首部的颜色色块，进行点对点吸取消除，并扣减毛线球点数。
* **目标：** 在槽位被占满前，消耗掉整条大龙。

### 1.2 底层逻辑抽象

* **机制本质：** 异步双端队列匹配。左端（Slot）为供应方，右端（Dragon Queue）为需求方。
* **风险点：** “槽位死锁”。即槽位已满，但大龙首部的颜色需求在槽位中无匹配，导致逻辑阻塞。

---

## 第二阶段：重构与开发建议

* **平台适配：** 针对移动端/小游戏，强化点击后的抛物线入槽动效与物理震动反馈。
* **复杂度控制：** 采用“伪随机”生成算法，确保棋盘上的毛线球总点数 $\sum Ball.Value \ge Dragon.Length$。
* **核心策略：** 强化玩家对“出球顺序”的规划，从单纯的反应力游戏转变为轻策略益智。

## 第三阶段：功能开发策划案 (GDD)

### 1. 功能目标

设计一套基于毛线球点数消耗的队列消除系统，实现点击、缓存、数值匹配扣减及大龙缩短的逻辑闭环。

### 2. 玩法逻辑流程 (Text-Map)

```text
[棋盘区] -> 玩家点击毛线球 (Color:ID, Value:N)
    |
    v
[缓存槽检测] -> IF 槽位 < 7 -> 进入槽位队列 (Slot List)
    |        -> ELSE -> 触发“槽满”警告反馈
    |
[实时匹配引擎] (轮询检测)
    |
    |--- 判定条件：Slot[i].Color == Dragon.Head.Color
    |
    |      YES (匹配成功)
    v      ----------------------> [吸取动画] 色块飞入毛线球
[数值结算] 
    |--- 1. Dragon.Head 色块销毁，大龙整体前移
    |--- 2. Slot[i].Value - 1
    |
    |--- IF Slot[i].Value <= 0
           |--> [销毁反馈] 毛线球消失，槽位后续球向左补位

```

### 3. 模块拆分

* **GridManager (棋盘模块):** 负责毛线球的静态分布、点击交互屏蔽（层级遮挡）。
* **SlotSystem (缓存槽模块):** 维护长度为 7 的对象数组，处理入槽平移及销毁补位逻辑。
* **DragonSystem (大龙模块):** 维护颜色色块序列，处理消除后的补位位移及路径移动。
* **MatchEngine (匹配引擎):** 核心控制器，处理 Slot 与 Dragon 之间的颜色比对与消除指令下发。

### 4. 详细子模块规则设计

#### A. 匹配与吸取规则

* **前端判定：** 大龙队列仅**最靠近消除口**的一个色块处于“可被吸取状态”。
* **优先级逻辑：** 若槽位中存在多个符合当前大龙首部颜色的球，优先消耗**靠左（索引在前）**的球。
* **连续吸取：** 若大龙前几个色块颜色相同，而对应槽位球点数充足，支持一次性快速吸取多个色块。

#### B. 毛线球与槽位规则

* **属性：** 包含 `ColorID`（颜色）和 `Value`（数值/耐久度）。
* **槽位移动：** 新球进入时，必须按照点击顺序排在当前最右侧球的右边；球销毁后，其右侧所有球立即向左对齐。
* **不可逆性：** 毛线球一旦入槽，除非消耗殆尽或使用特殊道具（如“移回棋盘”），否则不可移出。

#### C. 大龙移动规则

* **推进逻辑：** 大龙沿预设路径（Path）匀速前进。
* **击退/暂停：** 每次消除成功时，大龙可根据配置产生轻微的停顿或向起点方向的“击退”表现，缓解玩家压力。

### 5. 视觉表现规则 (VFX Specs)

| 阶段 | 视觉/音效表现 | 触发条件 |
| --- | --- | --- |
| **入槽** | 毛线球沿弧线轨迹由棋盘飞入槽位，落地伴随轻微Q弹动效。 | 点击棋盘可用球。 |
| **吸取** | 大龙首块变小并沿直线快速飞入目标槽位球中心。 | 匹配检测成功。 |
| **扣减** | 毛线球数字跳动，伴随轻微缩放（Scale）震动。 | 色块抵达毛线球中心。 |
| **消耗** | 毛线球化为丝线/碎片消失，并产生清脆的破碎音效。 | Value 归零。 |
| **失败** | 槽位边框变红、剧烈抖动，提示无法入球。 | 槽位满且无匹配。 |

### 6. 极端情况处理

* **快速连点：** 系统建立输入缓冲。即便玩家点击频率极高，球也必须按固定间隔依次入槽，防止动画重叠。
* **动态生成：** 关卡算法需在后台进行逆向校验，确保在当前毛线球分布下，至少存在一种消除序列能清空大龙。

### 7. 可扩展性与数值方向

* **数值逻辑：** * 随着关卡等级提升，增加单球的 `Value` 初始值（如从 3 提升至 10）。
* 增加大龙队列的颜色总数 $N$（如从 3 色增加至 8 色）。


* **系统扩展：** 预留“多头龙”接口（多个大龙共享一个槽位）及“特殊障碍球”接口。
